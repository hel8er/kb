## Матчасть

Стандарт [IEEE 802.1Q](https://ru.wikipedia.org/wiki/IEEE_802.1Qhttps://ru.wikipedia.org/wiki/IEEE_802.1Q) — это ключевой протокол, который позволяет создавать виртуальные локальные сети (VLAN) в Ethernet-сетях. Он был разработан для улучшения управления сетевым трафиком и повышения безопасности, позволяя разделять одну физическую сеть на несколько логических.

### Что такое VLAN?

[VLAN (Virtual Local Area Network)](https://ru.wikipedia.org/wiki/VLAN) — это логическая подгруппа в сети, которая объединяет устройства, независимо от их физического расположения. Это позволяет администраторам сети сегментировать трафик, улучшать производительность и обеспечивать безопасность, изолируя различные группы пользователей или устройств.

### Как работает 802.1Q?
![802.1Q Frame](https://upload.wikimedia.org/wikipedia/commons/0/0e/Ethernet_802.1Q_Insert.svg)
802.1Q добавляет специальный тег к Ethernet-кадрам, который содержит информацию о VLAN. Этот тег включает:

- **TPID (Tag Protocol Identifier)**: 2 байта, указывающие, что кадр содержит тег VLAN. Обычно это значение равно 0x8100.
- **TCI (Tag Control Information)**: 2 байта, содержащие информацию о VLAN, включая:
  - **Priority Code Point (PCP)**: 3 бита, определяющие приоритет трафика.
  - **Drop Eligible Indicator (DEI)**: 1 бит, указывающий, может ли кадр быть отброшен в случае перегрузки.
  - **VLAN Identifier (VID)**: 12 бит, определяющие номер VLAN (от 0 до 4095).

### Преимущества использования 802.1Q

1. **Сегментация сети**: Позволяет разделять трафик между различными группами пользователей, что улучшает управление и безопасность.
2. **Управление трафиком**: С помощью тегирования можно приоритизировать трафик, что особенно важно для приложений, чувствительных к задержкам, таких как VoIP.
3. **Упрощение администрирования**: VLAN позволяют легче управлять сетевыми ресурсами и изменениями в конфигурации сети.

## Пример конфигурации на MikroTik

1. Первое что небходимо сделать это включить фильтрацию VLAN на бридже

![[assets/bridge-vlan-filtering.png]]

В поле Frame types можно выставить тип трафика который будет обработан:
- **admit all** - любой
- **admit only untagged and priority tagged** - трафик в котором нет тега vlan и есть теги приоритетов
- **admit only vlan tagged** - только трафик с тегом vlan 

2. Далее мы можем пометить нетегированный трафик каждого из портов, установив сответствующий PVID(Port VLAN id). Таким образом, коммутация будет только между портами с одинаковым pvid. Данный тип портов, где разрешен только нетегированный трафик называется.**"Access port"**(Порт доступа) и используется, в основном, для подключения клиентских устройств.

![[assets/bridge-pvids.png]]

_Данный пункт можно опустить и метить нетегированный трафик в шаге 3 но лично мне этот способ удобнее и нагляднее_

3. Во вкладке Bridge -> Vlan у нас уже есть сгенерированные правила на основе PVID, установленных в шаге 2. Теперь поработаем с тегированным трафиком(если хочется то и нетегированным).

![[assets/bridge-vlan.png]]

Ключевой момент заключается в том чтобы при создании нового "правила" обязательно добавить бридж в тегированные интерфейсы, а затем уже в разделах **tagged** и **untagged** добавлять наши физические интерфейсы, на которые указанные vlan хотим назначить. На скрине выше у нас настроен Trunk(магистральный порт) Который принимает на себя vlan1 без тега и vlan10,11,90,111,120 с тегом, порты 2-5 настроены как порты доступа во vlan1, порт sfp1 настроен как порт доступа во vlan111.
