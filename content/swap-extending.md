Конечно! Обычно swap-файл создается в корневом каталоге и называется `swapfile`. Вот обновленные команды с использованием `/swapfile` в качестве имени файла:

### Увеличение swap-файла

1. **Проверьте текущий swap-файл:**
   ```bash
   sudo swapon --show
   ```

2. **Отключите текущий swap-файл:**
   ```bash
   sudo swapoff /swapfile
   ```

3. **Измените размер swap-файла:**
   Например, чтобы увеличить его до 4 ГБ:
   ```bash
   sudo dd if=/dev/zero of=/swapfile bs=1G count=4
   ```

4. **Установите правильные права доступа:**
   ```bash
   sudo chmod 600 /swapfile
   ```

5. **Создайте swap-файл:**
   ```bash
   sudo mkswap /swapfile
   ```

6. **Включите swap-файл:**
   ```bash
   sudo swapon /swapfile
   ```

7. **Проверьте, что swap включен:**
   ```bash
   sudo swapon --show
   ```

8. **Добавьте swap-файл в `/etc/fstab`, чтобы он автоматически подключался при загрузке:**
   Откройте файл `/etc/fstab` в текстовом редакторе и добавьте строку:
   ```
   /swapfile none swap sw 0 0
   ```

Теперь ваш swap-файл будет называться `/swapfile`. Не забудьте заменить `count=4` на нужное вам значение, если вы хотите другой размер.

Параметр `count=4` в команде `dd` указывает количество блоков, которые будут записаны. В данном случае, поскольку мы используем `bs=1G` (размер блока 1 гигабайт), `count=4` означает, что будет создано 4 блока по 1 ГБ каждый.
