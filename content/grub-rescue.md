---
title: Восстановление GRUB через SystemRescue
tags:
  - linux
---



1. В терминале запустите:
   ```bash
   lsblk
   ```
2. Найдите раздел с типом `ext4` (или `xfs`, `btrfs` и т.п.) и большим размером — это, скорее всего, корневая файловая система.
3. Убедитесь, что в этом разделе **используется значительный объём** (колонка "USED" не пустая).
4. Раздел со словом `swap` — это раздел подкачки, его монтировать не нужно.
5. Раздел с файловой системой типа `grub2 core.img`, отмеченный как `bios_grub`, — это технический раздел, его монтировать не нужно.
6. После определения, монтируйте корневой раздел (например, `/dev/sda2`).

---

## 1: Монтируем корневую файловую систему

```bash
mount /dev/sda2 /mnt
```

---

## 2: Монтируем системные каталоги

```bash
mount --bind /dev /mnt/dev
mount --bind /proc /mnt/proc
mount --bind /sys /mnt/sys
```

---

## 3: Переходим в установленную систему

```bash
chroot /mnt
```

---

## 4: Устанавливаем GRUB (BIOS режим)

```bash
grub-install /dev/sda
update-grub
```

---

## 5: Выход и перезагрузка

```bash
exit
umount -R /mnt
reboot
```

---

Если после перезагрузки снова возникают проблемы — проверьте, появляется ли ошибка GRUB, и при необходимости выполните повторную установку.
